# AIチャット

右パネルのチャットで、今のファイルや選択範囲を踏まえて AI に聞けます。

## 何ができる？

| モード | できること | ファイル変更 |
|--------|------------|--------------|
| **Ask** | 説明・質問・整理 | しない |
| **Edit** | 変更案を出す | プレビュー → あなたが適用 |
| **Agent** | ツールで読み取り・提案・（制限付き）実行 | 同上（詳細は [Agent](agent.md)） |

Ask / Edit / Agent とは別に、用途プリセット（一般 / 文書 / データ / コード）があります。話し方・方針の軸です。

| 用途 | Agent で増えやすいこと |
|------|------------------------|
| **文書** | Markdown を見出し単位で読む、見出し／相対リンクの軽い検証 |
| **データ** | CSV / TSV / JSON の列プロファイルと読み取り専用クエリ |
| **コード** / **一般** | 標準ツール（読取 / 検索 / 提案 / exec / verify） |

## 開き方

1. [AIを設定](../getting-started/ai-provider.md)する
2. （推奨）[フォルダを開く](../getting-started/open-project.md)
3. 右のチャットに入力して送信

## 使い方

- **Enter** … 送信
- **Shift+Enter** … 改行
- 開いているファイル全文と、選択中のテキストがあればそれもコンテキストに載ります
- チャットへ**ファイル**をドロップすると、読み取り専用の参照として追加できます（**フォルダは不可**）:
  - **左のエクスプローラー**（ワークスペース内のファイル）
  - **OS のファイルマネージャ**（例: Windows エクスプローラー）— フォルダを開いているとき、ワークスペース外のファイルも可
- **エディタタブ**をチャットへドラッグしても参照にできます
- 応答はストリーミング表示
- 履歴はワークスペースの `.compass/chat-history.json` に残ります

### Edit の適用

1. AI が変更を提案する
2. 差分プレビュー（追加＝緑、削除＝赤）を確認
3. **適用** または拒否
4. 適用後はメッセージ横の **この適用を取り消す**、成功バー、編集メニュー / `Ctrl+Shift+Z`、または **AI適用の履歴** から戻せます。現在のチャット分だけ新しい順に戻すこともできます

自動では書き込みません。適用後にそのファイルを手で編集すると、取り消しは拒否されます。より新しい適用がある場合は、先にそちらを取り消す必要があります。

### タブ操作

チャットタブを右クリックすると **閉じる** / **他を閉じる** / **すべて閉じる** が使えます。最後のチャットタブを閉じるとチャットパネルが隠れます。

## よくある質問

**Q. 何も返ってこない / エラー**  
A. [よくあるエラー](../troubleshooting/common-errors.md) を確認。まずは API Key とモデル。

**Q. Agent と Edit の違いは？**  
A. Edit は1回の応答で提案。Agent は読み取りやコマンドを挟みながら複数ステップ → [Agent](agent.md)

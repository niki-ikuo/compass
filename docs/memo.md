- [x] タブの削除で全タブがなくなったらタブエリアを閉じる。また、何にもないタブエリアを開いておく

## 対応判断（忖度なし）

### 1. 空タブエリアを閉じる
**判断: やる（優先度・低）→ 対応済み（チャットタブ）**

- 一文目は望ましい挙動、二文目は現状の問題（空のタブ帯が残ったまま開いている）の指摘。同じ要件の裏表。
- エディタ `TabBar` は既に `openFiles.length === 0` で非表示。問題の本体はチャット側で、最終タブを閉じると空セッションを自動生成していたこと。
- **やり方（実装）:** 最終オープンチャットを閉じたら空タブを作らず `showChat: false`。パネル再表示・新規作成・履歴再開・コンポーザ挿入時に必要なら空セッションを用意。

### 2. 未保存のままアプリ終了で消える
**判断: やる（優先度・最高）→ 対応済み**

- SPEC は autosave off・明示保存のみ。その前提なら終了時ガードがないのは欠陥。
- フォルダ閉じには dirty 確認があるのにアプリ終了に無いなら、ユーザーは「保存したつもり」で落ちる。
- **やり方（実装）:** ウィンドウ `close` / メニュー終了をインターセプト。dirty があればネイティブダイアログ（保存して終了 / 保存しない / キャンセル）。オートセーブは入れない。

### 3. エクスプローラで F2 リネーム
**判断: やる（優先度・中）。コスパ良い。**

- コンテキストメニューのリネームは既にある。Delete ショートカットもあるのに F2 だけ無いのは不揃い。
- Windows エクスプローラ／VS Code の期待値。実装も既存 inline rename にキーを足す程度。
- **やり方:** エクスプローラフォーカス時かつ選択1件・ルート以外で F2 → 既存 rename の `inlineInput` を起動。入力中・メニュー表示中は無視。複数選択は単一選択に揃えるか無効。

### 4. 新規ファイルを Markdown にする
**判断: やらない（デフォルト固定はしない）。**

- Compass は「テキスト全般」の AI workspace。code / data 用途もある。`.md` 固定は押し付け。
- 既に Markdown テンプレート経路がある。新規の既定を `.txt` のままにしてテンプレで MD を取る方が筋が通る。
- どうしても寄せるなら「用途プリセット連動」（document → `.md`、code → 拡張子なし or `.txt`）まで。全ユーザー共通の MD 化はしない。
- **やり方（やるなら）:** `defaultNewTextFile` をプリセット依存にするか設定項目にする。デフォルト一括変更は避ける。

### 5. 新規作成時に拡張子以外を選択
**判断: やる（優先度・高〜中）。小さい変更で効く。**

- 名前だけ打ち直して Enter、が自然。拡張子ごと選択だと `.txt` を消しがち。
- Explorer / VS Code と同じ。
- **やり方:** inline 入力の mount / focus 時に `setSelectionRange(0, lastDotIndex)`（ドット無し・先頭ドットのみは全選択）。新規作成とリネームの両方に同じ規則を適用すると一貫する。

### 6. Word / Excel を外部アプリで開く
**判断: やる（優先度・中）。ただし「アプリ内ビューア」は作らない。**

- Office バイナリを Monaco に食わせるのは無意味で危険（文字化け・巨大・破損リスク）。
- 製品の中核はテキスト編集＋AI。Office 編集はスコープ外。
- **やり方:**
  1. `.doc(x)` / `.xls(x)` / `.ppt(x)` 等を「外部オープン」対象に分類。
  2. ダブルクリック／開くで `shell.openPath`（または OS 既定アプリ）。失敗時は明確なエラー。
  3. エクスプローラに「既定のアプリで開く」を常設してもよい（Office 以外のバイナリにも流用可）。
  4. アプリ内プレビューや変換は後回し（ROI 低い）。
